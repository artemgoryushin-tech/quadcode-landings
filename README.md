# Quadcode Landing Pages

Static landing pages for Quadcode partner products and campaigns. Pages live under **`vlp/`** and are self-contained (HTML, assets, optional small scripts).

## Pages

### Creator Program (`vlp/creator-program/`)

Landing for the content creator partnership: short-form video (TikTok, Reels, YouTube Shorts) about the white-label brokerage platform, with performance-based rewards.

### Ambassador Program (`vlp/ambassador-program/`)

Landing for the ambassador / referral partnership: introduce qualified brokerage leads, choose cash rewards or a branded brokerage offer.

**Locales:** English is served from **`index.html`** at the folder root. Other languages use the same layout in subfolders, each with its own **`index.html`**:

| Code | Path |
|------|------|
| `en` | `vlp/ambassador-program/index.html` |
| `ru` | `vlp/ambassador-program/ru/index.html` |
| `pt` | `vlp/ambassador-program/pt/index.html` |
| `es` | `vlp/ambassador-program/es/index.html` |
| `th` | `vlp/ambassador-program/th/index.html` |
| `ko` | `vlp/ambassador-program/ko/index.html` |
| `vi` | `vlp/ambassador-program/vi/index.html` |

The in-page language switcher (markup aligned with [quadcode.com](https://quadcode.com/) header styles: `lang svelte-bvm2rm`, `lang_animation`, `lang_active`) links to these paths. Asset URLs on localized pages use **`../assets/`** so they resolve against each locale’s `<base href="https://quadcode.com/vlp/ambassador-program/{locale}/">`.

#### Regenerating localized HTML

From the repo root (Node 18+):

```bash
node vlp/ambassador-program/i18n/build.mjs
```

This reads **`vlp/ambassador-program/index.html`** (source of truth for English + switcher), then writes **`ru/`**, **`pt/`**, **`es/`**, **`th/`**, **`ko/`**, **`vi/`** `index.html` files with:

- replaced copy via string pairs (longest keys first),
- rebuilt language switcher for each locale,
- `<html lang="…">`, `<base>`, `canonical`, and `og:url` set per locale,
- `./assets/` → `../assets/` (and bare `assets/` → `../assets/`) where needed.

**Where strings live**

- **`i18n/strings.ru.mjs`** — Russian: array of `[englishFragment, russianHtml]` pairs.
- **`i18n/pt.json`** — Portuguese: JSON array of `[from, to]` pairs (same English keys as other JSON locales).
- **`i18n/es.json`**, **`th.json`**, **`ko.json`**, **`vi.json`** — generated by **`i18n/compose-locales.mjs`** (uses the English keys from **`pt.json`** and parallel arrays of translated strings inside that script). After editing **`pt.json`**, run:

  ```bash
  node vlp/ambassador-program/i18n/compose-locales.mjs
  node vlp/ambassador-program/i18n/build.mjs
  ```

- **`i18n/translations.mjs`** — wires `ru` plus the JSON locale files into **`replacements`** for `build.mjs`.

**Typical workflow**

1. Edit English copy and/or layout in **`vlp/ambassador-program/index.html`**.
2. Update **`strings.ru.mjs`** and/or **`pt.json`** (then re-run **`compose-locales.mjs`** if `pt.json` changed).
3. Run **`build.mjs`** and commit the updated locale **`index.html`** files.

### Trading Dashboard for Business (`vlp/td-business/`)

Landing for **Trading Dashboard for Business**: white-label market charts for brokers, trading platforms, and fintech apps (embeddable charting, brand customization, demo / lead flow). Single-language build; assets under **`vlp/td-business/assets/`** (fonts, hero imagery, logos, OG image, favicon).

## Repository layout

```
vlp/
├── creator-program/
│   ├── index.html
│   └── assets/
├── ambassador-program/
│   ├── index.html          # English (source for i18n build)
│   ├── ru/ … vi/           # Localized index.html per locale
│   ├── i18n/               # build + translation data (see above)
│   └── assets/
└── td-business/
    ├── index.html
    └── assets/
```

Each landing is deployable as its own static tree (path prefix e.g. `/vlp/<name>/` on the main site).

## Deployment

Subdirectories under **`vlp/`** can be deployed independently (Vercel, Netlify, static hosting, etc.). Match **`base href`** and canonical URLs to the environment (see **`.gitlab-ci.foach.yml`** for an example that rewrites `quadcode.com` → `quadcode2.foach.site` in `<base>` tags on the foach test host).
